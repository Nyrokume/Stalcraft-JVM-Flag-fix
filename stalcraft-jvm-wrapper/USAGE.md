# STALCRAFT JVM Wrapper — руководство по использованию (Tauri)

## Что реализовано

1. **Определение системы** — объём ОЗУ, ядра/потоки CPU, L3, проверка large pages и привилегии `SeLockMemoryPrivilege`.
2. **Генерация JVM-параметров** — по тем же правилам, что и в Go-версии: размер heap по таблице от **установленной** RAM (2–8 ГБ), потоки G1, регионы G1, JIT-профиль с расширением для CPU с большим L3 (условный «X3D-класс»).
3. **IFEO** — установка / удаление / проверка отладчика для `stalcraft.exe` и `stalcraftw.exe`.
4. **Профили конфигурации** — JSON в каталоге `configs/` рядом с exe; активное имя в `HKCU\Software\StalcraftWrapper\ActiveConfig`.
5. **Пресеты** — быстрые варианты поверх авто-профиля (см. ниже).
6. **Перехват запуска** — в режиме IFEO обёртка получает аргументы лаунчера, фильтрует чужие JVM-флаги, добавляет свои и поднимает приоритеты процесса игры.

## Рекомендуемый сценарий (IFEO)

1. Запустите приложение от имени администратора.
2. Нажмите **Refresh** в блоке HARDWARE PROFILE (обновит детекцию и список профилей).
3. При необходимости выберите **Preset** или профиль в **Saved profiles** и примените (см. раздел «Конфигурация»).
4. В блоке IFEO нажмите **INSTALL**.
5. Запускайте игру через EXBO / Steam и т.д. как обычно.

Прямой кнопки «Launch game» в текущем GUI нет: рабочий путь — IFEO и обычный запуск из лаунчера.

## Конфигурация: пресеты и сохранённые профили

### Где лежат данные

- Файлы: `configs/<имя>.json` рядом с `wrapper.exe`.
- Активный профиль: реестр `HKCU\Software\StalcraftWrapper\ActiveConfig` (строка = имя файла без `.json`).

### Кнопки

| Элемент | Действие |
|--------|----------|
| Чипы пресетов (Balanced, Latency, Throughput, Nursery, Conservative, Low RAM, Streaming, Power) | Пересчитать профиль от текущего железа, сохранить в соответствующий `preset_*.json`, установить активным |
| **Saved profiles** (список) + **Apply** | Только сменить активный профиль на выбранный JSON |
| **Regen** | Пересоздать `default.json` из детекции и сделать активным `default` |
| **Refresh** (в HARDWARE) | Обновить железо, целевой heap в UI и перечитать список `configs/*.json` |
| **Profile JSON editor** | **Load active** / **Load selected** — подставить JSON в поле; **Save to selected** — записать файл профиля из списка (имена полей как в `Config` в Rust) |

### Смысл пресетов (кратко)

- **Balanced** — как чистый `generate()` по железу.
- **Latency** — сильнее упор на низкие паузы GC.
- **Throughput** — более мягкий pause budget и чуть выше IHOP.
- **Nursery** — сдвиг в сторону большего young-gen.
- **Conservative** — без large pages и без pre-touch, более осторожные лимиты JIT.
- **Low RAM** — heap на 1 ГБ меньше авто (не ниже 2 ГБ).
- **Streaming** — меньше heap, без pre-touch, мягче GC, больше metaspace (захват/фон).
- **Power** — +1 к лимитам GC-потоков (в пределах капов), больше code cache.

## Heap и GC (соответствие коду)

### Heap (функция `size_heap`)

| Установленная RAM | Heap |
|-------------------|------|
| ≥ 24 ГБ | 8 ГБ |
| ≥ 16 ГБ | 6 ГБ |
| ≥ 12 ГБ | 5 ГБ |
| ≥ 8 ГБ | 4 ГБ |
| ≥ 6 ГБ | 3 ГБ |
| иначе | 2 ГБ |

### GC-потоки

- `ParallelGCThreads` = `clamp(threads − 2, 2, 10)`
- `ConcGCThreads` = `clamp(parallel / 2, 1, 5)` (в коде половина parallel, не «четверть»)

### G1 (ориентиры)

Базовые значения зависят от детекции (в т.ч. ветка для большого L3). Точные числа смотрите в сгенерированном `configs/*.json` или в исходнике `config.rs` / `jvm.rs`.

## IFEO — цепочка запуска

```
Лаунчер → stalcraft.exe → IFEO → wrapper.exe → игра с отфильтрованными/добавленными JVM-флагами
```

## Типичные проблемы

| Симптом | Что сделать |
|--------|-------------|
| Ошибка при INSTALL | Запуск от администратора |
| Не тот профиль | Проверить **Active** в блоке Configuration и список `configs/` |
| Нужен «чистый» авто-профиль | **Regen** для `default.json` |

Подробности по установке каталога `jvm_wrapper` и исключениям антивируса — в [README.md](./README.md).
